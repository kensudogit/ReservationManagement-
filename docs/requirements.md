# 要件定義書 — 撮影スタジオサブスク予約管理システム

## 1. プロジェクト概要

撮影スタジオのサブスクリプション会員向けに、スタジオ枠の予約・確認・検索・キャンセル、および Google カレンダー連携を提供する Web アプリケーション。

| 項目 | 内容 |
|------|------|
| システム名 | Studio Reservation Manager (SRM) |
| 対象ユーザー | サブスク会員 / スタジオ管理者 |
| 依頼工程 | 要件定義 / 設計 / フロントエンド / バックエンド / 保守・運用 |

## 2. ステークホルダー

| ロール | 権限 |
|--------|------|
| 会員 (member) | 自分の予約の作成・確認・検索・キャンセル |
| 管理者 (admin) | 全予約・スタジオ・会員の管理、枠設定 |

## 3. 機能要件

### 3.1 予約確認機能
- ログインユーザーの予約一覧を表示（日時・スタジオ・ステータス）
- 予約詳細（枠時間、メモ、カレンダー同期状態）を確認できる
- 管理者は全会員の予約を確認できる

### 3.2 検索機能
- 日付範囲・スタジオ・ステータス・キーワードで予約を検索
- 空き枠検索（スタジオ × 日付）で予約可能なスロットを表示

### 3.3 キャンセル機能
- 会員は自分の予約をキャンセル可能（キャンセルポリシー: 開始 24 時間前まで）
- 管理者は任意の予約をキャンセル可能
- キャンセル時に Google カレンダー上のイベントも削除（連携済みの場合）

### 3.4 Google カレンダー連携
- OAuth 2.0 で Google アカウントを接続
- 予約作成・更新・キャンセル時にカレンダーイベントを同期
- 連携状態の表示・解除が可能

### 3.5 その他（MVP に含む）
- 会員登録 / ログイン（JWT）
- スタジオ・利用可能枠のマスタ管理（管理者）
- サブスクプラン情報の表示（会員の残枠数は簡易実装）

## 4. 非機能要件

| 項目 | 目標 |
|------|------|
| 可用性 | 開発環境は Docker Compose で再現可能 |
| セキュリティ | パスワードは bcrypt ハッシュ、API は JWT 認証 |
| パフォーマンス | 一覧 API はページネーション対応 |
| 保守性 | 要件・設計・運用ドキュメントを同梱 |

## 5. 技術スタック

| 層 | 技術 |
|----|------|
| フロントエンド | Next.js 15 (App Router) + React + TypeScript |
| バックエンド | Python 3.12+ / FastAPI / SQLAlchemy 2 / Alembic |
| DB | PostgreSQL 16 |
| 認証 | JWT (python-jose) + bcrypt |
| 外部連携 | Google Calendar API (OAuth 2.0) |
| インフラ | Docker Compose |

## 6. スコープ外だった項目（実装済みに更新）

- プラン変更の日割り (proration)
